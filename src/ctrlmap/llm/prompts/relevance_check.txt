## Rules
REJECT as not relevant if:
- **Boilerplate**: purpose/scope, approval blocks, headers, disclaimers. \
"This policy defines the requirements for controlling access" is NOT an enforcement \
mechanism — it describes what the policy covers without prescribing controls.
- **Domain mismatch**: physical security ↔ logical access, or unrelated domain.
- **Process mismatch**: different process sharing vocabulary.
- **Subject mismatch**: same action, different subject. "Reviewing user access rights" \
(operational) ≠ "reviewing/updating the access control policy" (governance about the \
document itself). When the control says "review and update the policy," the evidence must \
be about THE POLICY DOCUMENT — not about performing activities the policy describes.

ACCEPT as relevant if:
- Synonyms/paraphrases for the SAME concept: "credential management" = "authenticator \
management"; "password complexity" = "authenticator strength"; "security awareness training" = \
"security literacy training."
- Partial coverage: addresses at least ONE sub-requirement substantively.
- Same subject, different approach (e.g., "prohibited" vs "exception basis" — both address shared IDs).

### Rejection examples
Control: "Enforce approved authorizations for logical access."
Chunk: "This policy defines the requirements for controlling access to systems. It applies to all \
employees. The purpose is to ensure access is granted based on business need."
→ {{"relevant": false}} (boilerplate: purpose/scope only — no enforcement mechanisms)

Control: "Develop, document, and disseminate an access control policy. Review and update the policy."
Chunk: "Administrators review user access rights quarterly. Excessive privileges are removed."
→ {{"relevant": false}} (subject mismatch: reviewing access rights ≠ reviewing the policy document)

Control: "Developer testing — create a security test plan."
Chunk: "Network vulnerability scans performed weekly."
→ {{"relevant": false}} (process mismatch: network scanning ≠ developer testing)

### Acceptance examples
Control: "Authenticator management — ensure sufficient strength, refresh periodically."
Chunk: "Credential management enforces 14+ chars. Credentials expire every 90 days."
→ {{"relevant": true}} (synonym: credentials = authenticators; complexity = strength)

Control: "Security awareness training for all users."
Chunk: "All employees complete security education in week one."
→ {{"relevant": true}} (synonym: education = training)
